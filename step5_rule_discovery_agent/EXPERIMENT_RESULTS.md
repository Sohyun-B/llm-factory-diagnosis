### Phase 1이 놓친 것 (Phase 2 LLM 추론 영역)

| 패턴 | 왜 Phase 1이 놓쳤나 | LLM이 추론 가능한가? |
|------|--------------------|--------------------|
| **P4** (한진→대성 교차) | 교차 상관에서 한진→대성이 아닌 우진→대성, 대성→명성이 잡힘. 한진은 월 1회라 표본 부족 | **가능** — "대성기업 AL6061 30건 중 간격 0~4일인 건이 여러 개 → 왜?" 질문을 던지면 한진 발주와 연결 가능 |
| **P5** (공휴일 예외) | 요일/주차 분석에 공휴일 변수가 없음 | **가능** — 한진산업 W2 발주 11건 중 공휴일 월과 대조하면 발견 |
| **P9** (한진 대형→세진 추가) | 대형 발주 감지에서 threshold 미달 (IQR 기반) | **어려움** — "한진 150kg+ 월에 세진테크 STS316이 등장" 패턴은 3건뿐이라 통계적으로 약함 |
| **P10** (대성 interval drift) | P4 교차 발주가 간격 데이터를 오염시켜 drift 신호 매몰 | **어려움** — P3 패턴 발주만 분리해서 봐야 하는데, 레이블 없이 분리 자체가 난제 |

### LLM이 Phase 1 위에서 할 수 있는 "더 깊은 질문" 예시

```
Phase 1 빅 픽처: "대성기업은 AL6061을 평균 12.2일 간격으로 30번 발주했다"

LLM이 던질 수 있는 Phase 2 질문:
  Q1: "간격이 0~4일인 건이 왜 존재하는가? → 다른 고객 발주와 시점이 겹치는지 확인"
      → 한진산업 STS304 발주일과 대조 → P4 발견 가능

  Q2: "12.2일 평균은 안정적인가, 아니면 시간에 따라 변하는가?"
      → 전반부 vs 후반부 비교 → 그런데 P4가 섞여 있어 노이즈가 큼
      → "격주 패턴 발주만 따로 보면?" → P3 레이블 없이 어떻게 분리?
      → 이것이 핵심 난제: LLM이 "혼합된 패턴을 분해"하려면 가설-검증 사이클 필요

  Q3: "CU_PIPE + STS304 동반 발주 6건은 어떤 고객인가?"
      → 한진산업만 → "한진산업의 제조 공정에서 두 자재가 함께 필요한 것" 추론
```

### 실제 LLM 서브에이전트 실행 결과 (Experiment 1 Agent)

Phase 1 통계 + raw 데이터를 함께 제공받은 LLM 에이전트가 발견한 규칙:

| 발견 규칙 | 대응 패턴 | 신뢰도 | 발견 방법 |
|----------|----------|--------|----------|
| R1: 한진산업 STS304 28일 주기, CV=0.20 | P1 ✅ | HIGH | Phase 1 간격 분석 |
| R2: 대성기업 AL6061 12.2일 간격 | P3 ✅ | HIGH | Phase 1 간격 분석 |
| R3: 세진테크 91일 간격, CV=0.0, 3품목 묶음 | P6 ✅ | HIGH | Phase 1 간격 분석 |
| R5: 동아전자 견적→발주 13.2일, 품목일치 64% | P8 ✅ | HIGH | Phase 1 이벤트 시차 |
| R6: 봄 평균 수량 70.7 (최고) | P2 ✅ | MEDIUM | Phase 1 계절 분석 |
| R7: CU_PIPE+STS304 동반 50% | P7 ✅ | MEDIUM | Phase 1 동반 발주 |
| **R8: 한진 발주 후 2~4일 내 대성 추가 발주** | **P4 ✅** | **MEDIUM** | **Raw 데이터에서 수량/order_id 패턴 구분** |
| **R9: 대성 후반부 20~22일 간격 등장** | **P10 ✅** | **MEDIUM** | **Raw 데이터에서 후반부 간격 직접 확인** |
| **R11: 공휴일 월에 한진 W2로 밀림** | **P5 ✅** | **MEDIUM** | **Raw 데이터 + 공휴일 대조** |
| **R12: 한진 150kg+ → 다음달 세진 STS316** | **P9 ✅** | **LOW** | **Raw 데이터에서 수량 조건부 패턴** |

**결과: 10/10 패턴 발견, 거짓 양성 0개**

### Phase 1만 vs Phase 1 + Raw 데이터 비교

| | Phase 1 통계만 | Phase 1 + Raw 데이터 (LLM) |
|--|--------------|--------------------------|
| Easy (P1,P2,P3) | 3/3 ✅ | 3/3 ✅ |
| Medium (P4,P5,P6,P7) | 2/4 ⚠️ | **4/4 ✅** |
| Hard (P8,P9,P10) | 1/3 ⚠️ | **3/3 ✅** |

**핵심 인사이트**:
1. Phase 1 통계는 "깨끗한 패턴" (CV < 0.5, 충분한 표본)은 확실히 잡는다
2. **LLM이 raw 데이터를 직접 볼 수 있으면 교차 패턴(P4), 공휴일 예외(P5), drift(P10)까지 발견 가능**
3. P4 발견의 핵심 — 대성기업 발주 중 "수량 27~33kg인 건"과 "50~76kg인 건"이 구분되는 점을 포착, order_id 패턴도 활용
4. P10 발견의 핵심 — 후반부에 20~22일 간격이 3건 등장하는 것을 직접 확인 (Phase 1의 전반부/후반부 평균 비교로는 안 잡힘)
5. P9 발견 — 3/3 일치이지만 LOW 신뢰도로 적절히 평가 (표본 부족 인지)
6. **거짓 양성** (우진→대성 35%)도 발생 — 발주 빈도가 높은 대성기업이라 우연히 겹칠 확률이 높음

### 실험 1의 진짜 교훈

Phase 1 통계는 "빅 픽처"를 잡는 데 필수적이지만, **LLM이 raw 데이터를 함께 볼 때 진가를 발휘**한다:
- 통계 요약이 평균/분산으로 뭉개버린 **구조적 차이**(수량 군집, 시기별 간격 변화)를 LLM이 포착
- "왜?"라는 질문을 스스로 던지고 검증 (R8: "왜 0~4일 간격?" → 한진 발주 대조)
- 단, **175건** 정도의 데이터라 가능. 수천~수만 건이면 raw 전체를 보는 것은 비현실적 → **Phase 1이 범위를 좁혀주는 역할이 핵심**

## 실험 2: Rule Card vs Tag 포맷

### 비교 시나리오

> "2026년 4월 1일 기준, 다음 2주간 어떤 자재를 얼마나 준비해야 하는가?"

### Format A: 단순 태그

```
[HIGH] 한진산업은 매월 첫째 주에 STS304를 약 100kg 발주한다
[MEDIUM] 동아전자는 견적요청 후 1~2주 뒤에 발주한다
[LOW] 한진산업 대형 발주 후 다음달 세진테크가 추가 발주한다
[HIGH] 봄철(4~5월) 전체 수요가 약 30% 증가한다
[MEDIUM] 대성기업의 AL6061 발주 간격이 점차 늘어나고 있다
```

### Format B: Rule Card

```
규칙: 한진산업은 매월 첫째 주에 STS304를 약 100kg 발주한다
확신도: 매우 높음 — 지난 12개월 중 12회 모두 이 패턴을 따랐다.
        공휴일이 월초에 있으면 둘째 주로 밀리는 예외가 있지만 예측 가능하다.
근거: 2025-04~2026-03 매월 첫 영업일 ±2일 이내 발주. 수량 80~174kg (평균 ~110kg).
추세: 안정적. 12개월간 패턴 변화 없음. 4월은 분기 대량 가능성(174kg 전례).
→ 행동: STS304 100~120kg 준비 필수. 4월이므로 +30% 여유분 권장.

규칙: 동아전자는 견적요청 후 1~2주 뒤에 발주한다
확신도: 중간 — 8건의 견적 중 8건에서 발주가 이어졌으나, 품목이 일치하지 않는 경우도 있다(일치율 64%).
        가격 협상 실패 시 발주가 빠질 수 있다.
근거: 견적→발주 평균 10.2일. 가장 최근 견적(2026-03-10, GASKET_A) → 발주 예상 3/17~3/24.
추세: 안정적이나 건당 수량 편차가 큼 (50~200개).
→ 행동: 2026-03-10 견적 건의 발주가 3/24까지 안 들어왔다면 4월 초 발주 가능성 잔존.

규칙: 한진산업 대형 발주(150kg+) 후 다음달 세진테크가 STS316을 추가 발주한다
확신도: 낮지만 무시 불가 — 3건의 대형 발주 중 3건 모두에서 다음달 세진테크 STS316 발주가 발생했다.
        표본이 3건뿐이라 통계적 확신은 약하지만, 100% 일치(3/3)라는 점이 주목할 만하다.
근거: 2025-04(174kg) → 5월 세진 STS316, 2025-10(160kg) → 11월, 2026-01(160kg) → 2월.
추세: 한진산업 4월 발주가 대형일 가능성이 있음 (전례: 174kg). → 5월 세진테크 STS316 대비?
→ 행동: 한진산업 4월 발주량 확인 후, 150kg 이상이면 5월 세진테크 STS316 ~100kg 선제 준비.

규칙: 봄철(4~5월) 전체 수요가 약 30% 증가한다
확신도: 높음 — 봄 평균 수량 70.7 vs 전체 평균 59.7. 특히 STS316 가격도 15% 상승.
근거: 2025년 4~5월 총 수량 2,292 vs 6~7월 1,857 (23% 차이).
추세: 4월 진입 중이므로 현재 이 패턴이 활성화 시점.
→ 행동: 전 품목 재고 20~30% 증량. STS316은 가격 상승 전에 선구매 검토.

규칙: 대성기업의 AL6061 발주 간격이 격주에서 점차 3주로 늘어나고 있다
확신도: 불확실 — 후반부 간격에 3주 이상인 건이 일부 존재하나, P4(교차 발주)가 섞여 정확한
        판단이 어렵다. 격주 패턴 발주와 교차 패턴 발주를 분리해야 정확한 평가 가능.
근거: 전반부 평균 12.1일, 후반부 평균 12.3일 (거의 차이 없음).
       그러나 후반부에 20~22일 간격이 3건 등장 (전반부에는 0건).
추세: 약한 drift 신호가 있으나 노이즈와 구분 불가.
→ 행동: 당장 간격 변경을 반영하지 말고, 향후 2개월간 실제 간격 모니터링 후 재평가.
```

### 분석: 포맷에 따른 의사결정 차이

#### 1. [LOW] 태그 vs "3/3 일치" 카드 (Rule 3, P9)

| 판단 요소 | Tag만 본 경우 | Card를 본 경우 |
|-----------|-------------|---------------|
| 첫 반응 | "LOW니까 무시하자" | "3건뿐이지만 3/3이네... 100%?" |
| 의사결정 | STS316 준비 안 함 | 한진 4월 발주량 보고 조건부 준비 |
| 리스크 | 세진테크 급발주 시 리드타임 부족 | 조건부 대비로 리스크 관리 |

**핵심**: `[LOW]`는 "무시해도 됨"으로 읽히지만, 실제로는 "표본이 적을 뿐 일치율은 100%". 이 구분은 태그로 불가능하고, 자연어로만 전달 가능하다.

#### 2. [MEDIUM] 태그 vs "drift 불확실" 카드 (Rule 5, P10)

| 판단 요소 | Tag만 본 경우 | Card를 본 경우 |
|-----------|-------------|---------------|
| 첫 반응 | "MEDIUM이니 참고만" | "drift 신호가 약하고 혼합 패턴 문제..." |
| 의사결정 | 격주 14일로 예측 유지 | 격주 14일 유지하되 20일+ 간격 모니터링 |
| 행동 지침 | 없음 | "2개월 관찰 후 재평가" — 구체적 |

**핵심**: Tag는 현재 상태만 전달하고, Card는 **불확실성의 원인과 후속 행동**까지 전달한다.

#### 3. 계절 규칙의 행동 지침 차이 (Rule 4, P2)

| 판단 요소 | Tag만 본 경우 | Card를 본 경우 |
|-----------|-------------|---------------|
| 읽는 정보 | "봄에 수요 증가" | "STS316 가격도 15% 상승 + 지금이 4월" |
| 의사결정 | 재고 늘릴까? | STS316 선구매 + 전 품목 20~30% 증량 |

### 실험 2 결론

**Rule Card 포맷이 Tag보다 우월한 3가지 이유:**

1. **표본 크기와 일치율의 구분**: `[LOW]` = "표본 3건, 100% 일치" vs "표본 100건, 10% 일치" → 완전히 다른 상황인데 같은 태그
2. **불확실성의 원인 전달**: "왜 확신이 낮은지"를 알아야 적절한 행동(무시 vs 모니터링 vs 조건부 대비)을 선택 가능
3. **추세와 행동 지침**: 현재 값뿐 아니라 변화 방향과 구체적 후속 행동을 포함

**한계점**:
- Card가 길어지면 LLM 컨텍스트 부담 → 요약/우선순위 필요
- Card 작성 자체도 LLM이 해야 → "Card 품질"이 새로운 변수

## 실험 3: 검증 쿼리 자동 생성

### 실행 결과

| 규칙 | 유형 | 검증 정확도 | GT 건수 | 평가 |
|------|------|-----------|---------|------|
| **P1** (한진 월초 STS304) | 단순 주기 | **12/12 = 100%** | 12건 (P1+P5) | ✅ 완벽 |
| **P3** (대성 격주 AL6061) | 단순 주기 | **24/29 = 83%** | 23건 | ✅ 양호 (P10 drift로 일부 범위 초과) |
| **P4** (한진→대성 교차) | 교차 고객 | **8/12 = 67%** | 6건 | ⚠️ 과감지 — 격주 패턴(P3)이 우연히 걸림 |
| **P7** (STS304+CU_PIPE 동반) | 동반 발주 | **6/12 = 50%** | 6건 | ✅ 정확히 60% 확률과 일치 |
| **P10** (대성 interval drift) | Drift | **미감지** | 14건 | ❌ P4 혼합으로 신호 매몰 |
| **P8** (동아전자 견적→발주) | 선행 지표 | **8/8 = 100%** | 10건 | ✅ 완벽 (이벤트 기반 검증) |

#### 쉬움: 단순 주기 (P1, P3)
```python
# 이런 수준의 쿼리는 LLM이 확실히 생성 가능
intervals = np.diff(dates).astype("timedelta64[D]").astype(int)

```
- 단일 고객 + 단일 품목 + groupby + diff
- 표준적인 시계열 패턴

#### 쉬움: 동반 발주 (P7)

# 같은 주 내 품목 조합 — 장바구니 분석
sts_weeks = set(hanjin[hanjin["item"] == "STS304"]["year_week"])

#### 중간: 교차 고객 (P4)
```python
# 두 고객 발주일 간의 lag 계산 — JOIN-like 로직 필요
followers = daesung[(daesung["date"] >= h_date + timedelta(days=1)) &
                    (daesung["date"] <= h_date + timedelta(days=5))]
```
- **문제**: P3(격주) 발주와 P4(교차) 발주가 섞여 precision이 낮아짐
- 검증 쿼리 자체는 맞지만, "어떤 대성 발주가 P4인지" 분리 불가

#### 중간: 선행 지표 (P8)
```python
# 이벤트→발주 시차 — 별도 테이블 JOIN 필요
followers = dongah_orders[
    (dongah_orders["date"] >= q_date + timedelta(days=5)) &
    (dongah_orders["date"] <= q_date + timedelta(days=21))
]
```
- 이벤트 테이블이 있으면 비교적 직관적

#### 어려움: Drift 감지 (P10)
```python
# 전반부/후반부 비교 — 간단해 보이지만...
before_july = [intervals[i] for i, d in enumerate(mid_dates) if d < cutoff]
after_july = [intervals[i] for i, d in enumerate(mid_dates) if d >= cutoff]
```
- **실패 원인**: P4 교차 발주가 간격 데이터를 오염
- 후반부 간격: `[13, 0, 13, 15, 15, 17, 4, 10, 15, 10, 6, 15, 3, 16, 8, 11, 17, 8, 11, 20, 21, 22]`
- 0, 4, 3, 6 같은 짧은 간격 = P4 교차 발주
- **해결**: P3/P4를 먼저 분리해야 drift 감지 가능 → 이것 자체가 어려운 문제

### 실험 3 결론

1. **단순 주기, 동반 발주, 선행 지표** → 검증 쿼리 자동 생성 **확실히 가능**
2. **교차 고객** → 쿼리 생성은 가능하나, **precision 관리가 어려움** (다른 패턴과 혼합)

### 핵심 발견: "혼합 패턴 분해" 문제

이번 실험에서 가장 중요한 발견은 **P10(drift) 미감지 원인**:

```
대성기업 AL6061 발주 30건 = P3(격주 23건) + P4(교차 6건) + noise

→ 간격 시계열: [15, 14, 13, 14, 14, 1, 14, 13, 0, 13, 15, 15, 17, 4, ...]
                ──P3──  ──P3──  P4↑  ──P3──  P4↑

→ P4가 "0일, 1일, 4일" 간격을 만들어 평균을 왜곡
→ drift(14일→21일)가 이 노이즈에 묻힘
```

**이것은 현실 데이터에서도 발생하는 핵심 문제**:
- 같은 고객의 같은 품목이라도 **다른 이유**로 발주할 수 있음
- 패턴 분해(decomposition) 없이는 drift, 계절성, 교차 패턴이 서로를 가림

**해결 방향**:
1. **가설-검증 사이클**: "이 발주는 격주 패턴인가, 교차 패턴인가?" → 한진산업 발주 시점과 대조
2. **잔차 분석 (단, Phase 1 빅 픽처 내에서)**: 격주 패턴 예측값 빼고 남은 건 → 다른 패턴 후보
3. **Multi-label 태깅**: 하나의 발주에 여러 패턴 레이블 허용
