# Step 5: Rule Discovery Agent

## ML-LLM 역할 분담 원칙

**커버리지는 ML이 보장, 깊이는 LLM이 결정**

LLM이 자율적이어야 하는 것:
- 패턴의 **해석** (이게 뭘 의미하는지)
- 불일치의 **원인 추론** (왜 안 맞는지)
- 가설의 **깊이** (얼마나 더 파야 하는지)

LLM이 자율적이면 안 되는 것:
- **무엇을 볼지** 선택 — LLM이 필터링하면 대부분의 신호가 소실됨

## 대화 원칙

- **단일 대화 유지** — 모든 LLM 턴은 하나의 대화(messages 누적)로 이어져야 함. 끊기면 맥락 소실.
- **LLM 출력은 항상 깨진다고 가정** — test_type 오타, 파라미터 누락, 존재하지 않는 도구 호출이 매번 발생함. normalize + fallback 필수.
- **Phase 1은 고정, 재실행 없음** — 기초 통계는 한 번만 돌리고 LLM이 해석만 함.
- **수사 흐름: 큰 그림 → 검증 → 불일치 추적 → 깊이 파기** — Phase 1/2/3 순서가 아니라 탐구의 논리적 흐름을 따름.
